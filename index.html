Introduction to React

React is a JavaScript library developed by Facebook for building user interfaces. It allows developers to create large web applications that can change data, without reloading the page.

One of the primary benefits of React is its ability to efficiently update and render just the right components when your data changes, thanks to its virtual DOM.

To start using React, you need to set up a React application using tools like Create React App. Here's a simple way to create a new React app:

npx create-react-app my-app

Once your project is set up, you can create your first component:

function MyComponent() {'{'}
return <div>Hello World!</div>
{'}'}

This is a basic introduction to React and its component-based architecture.

Components in React

Components are the building blocks of a React application. They encapsulate logic and UI in reusable pieces.

Components can be defined as either functional components or class components.

A functional component is a JavaScript function that returns a React element. Here’s a simple example:

const MyFunctionalComponent = () => {'{'}
return <h1>Welcome to React!</h1>
{'}'}

Class components, on the other hand, are ES6 classes that extend from React.Component:

class MyClassComponent extends React.Component {'{'}
render() {'{'}
return <h1>Welcome to React Class Component!</h1>
{'}'}
{'}'}

Components can accept inputs called props, which allow you to pass data from parent to child components.

  • Reusable code
  • Separation of concerns
  • Easy to test
  • Better maintainability
  • Improved collaboration
State and Props

State is a built-in object that allows components to create and manage their own data. It can be modified using the setState function in class components or the useState hook in functional components.

Props, on the other hand, are immutable and are used to pass data from a parent component to a child component.

Here’s a simple example of using state in a functional component:

const Counter = () => {'{'}
const [count, setCount] = useState(0);
return (
<div>
<p>Count: {count}</p>
<button onClick={() => setCount(count + 1)}>Increment</button>
</div>
);
{'}'}

This allows components to be dynamic and interactive, enhancing the user experience.

Lifecycle Methods

Lifecycle methods are hooks that allow you to run code at specific points in a component's life. They are primarily used in class components.

The most common lifecycle methods are:

  • componentDidMount: Invoked immediately after a component is mounted.
  • componentDidUpdate: Invoked immediately after updating occurs.
  • componentWillUnmount: Invoked immediately before a component is unmounted and destroyed.

Here's an example that uses lifecycle methods:

class MyComponent extends React.Component {'{'}
componentDidMount() {'{'}
console.log('Component mounted');
{'}'}
render() {'{'}
return <div>Hello World!</div>
{'}'}
{'}'}

Using these methods helps manage resources effectively in your React applications.

Hooks in React

Hooks are a feature introduced in React 16.8 that allow you to use state and other React features without writing a class.

The most common hooks are useState and useEffect.

Here's how you can use the useState hook:

const MyComponent = () => {'{'}
const [name, setName] = useState('');
return (
<input type="text" onChange={(e) => setName(e.target.value)} />
<p>Hello, {name}!</p>
);
{'}'}
